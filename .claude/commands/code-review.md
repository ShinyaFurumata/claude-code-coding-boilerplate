code-reviewer エージェントにレビューを依頼してください。差分の確認は不要で、レビュー範囲の指定のみエージェントに指示します。
エージェントからの結果を整理し、修正すべき箇所は<*なぜ*これが問題なのかを説明する有効な Markdown。ファイル/行/関数を引用>の形で伝えてください。
また、マージ可否の判断もユーザーに伝えてください。

エージェントへの指示はなるべく簡潔にしてください。エージェントはどのような結果を返すべきか理解しています。
`例： uncommit changesをレビューしてください。`

## レビュー範囲

ユーザー範囲指定がある場合は下記条件に従って範囲を伝えてください。

- `local`: uncommit changes のみ
- `develop`: develop ブランチとの差分 + uncommit changes
- 指定なし: develop ブランチとの差分 + uncommit changes

ユーザー指定範囲: #$ARGUMENTS
